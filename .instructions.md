# Project Instructions

## Project Overview

This is a monorepo containing multiple apps and libraries:

- **apps/** - Applications using the libraries
  - `recorder/` - Main recorder application
- **libs/** - Reusable libraries
  - `telemetry/` - Telemetry packet parsing and handling

## Project Management

This project uses `uv` as the main tool for project management, testing, linting, and type checking.

### Running Tests

- Use `uv run pytest` to run all tests
- Use `uv run pytest tests/` to run tests in a specific directory
- Use `uv run pytest <path>/<test_file>.py` to run a specific test file

### Linting and Formatting

- Use `uv run ruff check` to lint the code
- Use `uv run ruff format` to format the code
- Use `uv run ruff check --fix` to auto-fix linting issues

### Type Checking

- Use `uv run ty check` to run type checking across the project

## Test File Organization

Test files must be organized in the following structure:

- **Location**: `tests/` directory at the root of each library
- **Hierarchy**: Mirror the structure of the `src/` directory
- **Naming**: Prefix test files with `test_`

### Example Structure

```
libs/<library>/
├── src/
│   └── <library>/
│       ├── module1.py
│       ├── module2.py
│       └── packet/
│           ├── __init__.py
│           ├── car_damage.py
│           └── ...
└── tests/
    └── <library>/
        ├── test_module1.py
        ├── test_module2.py
        └── packet/
            ├── test_car_damage.py
            └── ...
```

## Coding Standards

1. **Type Annotations**: Use type hints for all functions and classes
2. **Testing**: Always include tests for new modules and features
3. **Code Quality**: Run `uv run ty check` before committing code
4. **Naming**: Follow Python naming conventions (snake_case for functions/variables, PascalCase for classes)

## Development Workflow

1. Create a new branch for your feature or fix
2. Make your changes
3. Add corresponding test files in `tests/`
4. Run `uv run ruff check --fix` to format and fix linting issues
5. Run `uv run ty check` to ensure type checking passes
6. Run `uv run pytest` to ensure all tests pass
7. Commit and push your changes
