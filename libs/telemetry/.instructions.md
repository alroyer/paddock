# Test File Management Instructions

## Test Structure

Test files must be organized according to the following structure:

- **Location**: `tests/` at the root of the project
- **Hierarchy**: Mirror the structure of the `src/` directory
- **Naming**: Prefix test files with `test_`

### Example Structure

```
libs/telemetry/
├── src/
│   └── telemetry/
│       └── packet/
│           ├── __init__.py
│           ├── car_damage.py
│           ├── car_setup.py
│           └── ...
└── tests/
    └── telemetry/
        └── packet/
            ├── test_car_damage.py
            ├── test_car_setup.py
            └── ...
```

## Conventions

1. **One test file per module**: Create `tests/telemetry/packet/test_<module_name>.py` to test `src/telemetry/packet/<module_name>.py`
2. **Missing directories**: Create the `tests/telemetry/packet/` directories if they don't exist
3. **`__init__.py` files**: Include `__init__.py` files in test directories

## When to Add Tests

- When creating a new module in `src/telemetry/packet/`
- When fixing a bug
- When adding a new feature

Always ensure that the corresponding test file exists in the correct location.

## Project Management Tools

This project uses `uv` as the main tool for project management and testing:

### Running Tests

- Use `uv run pytest` to run all tests
- Use `uv run pytest tests/telemetry/packet/test_<module_name>.py` to run specific tests

### Linting and Formatting

- Use `uv run ruff check` to lint the code
- Use `uv run ruff format` to format the code
- Use `uv run ruff check --fix` to auto-fix linting issues

### Type Checking

- Use `uv run ty check` to run type checking

### All Checks

- Use `uv run ty check` to run all linting, formatting, and type checking in one command
